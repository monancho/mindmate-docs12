# 프로젝트 개요

## 1. 서비스 개요

MindMate는 감정 기록을 기반으로 **AI 분석과 시각화 리포트**를 제공하는 감정 중심 자기관리 서비스입니다.
사용자가 일기를 작성하면 시스템이 감정 정보를 자동으로 분석해 **오늘의 상태·패턴·변화**를 한눈에 이해할 수 있도록 돕습니다.

---

## 2. 문제 정의 및 기획 배경

-   감정을 기록해도 **정확한 해석이 어렵다**
-   시간이 지나면 **감정 변화나 패턴을 확인하기 어렵다**
-   기존 자기관리 도구는 **할 일 중심**으로 감정·심리 관리 기능이 부족하다

!!!info "MindMate의 핵심 해결 방향"

    감정 텍스트에서 감정을 자동 추출하고, 분석 결과를 시각화하여<br>
    사용자가 스스로 감정 변화를 이해할 수 있도록 돕는 구조로 설계되었습니다.

---

## 3. 주요 기능

!!!tip "핵심 기능 요약"

    MindMate 서비스의 중심 기능은 **감정 일기**, **AI 감정 분석**, **시각화 리포트**입니다.

### 감정 일기

-   텍스트 기반 기록
-   감정 이모지 선택
-   이미지 업로드
-   날짜별 작성·수정·삭제

### AI 감정 분석

-   일기 내용에서 감정 범주 자동 추출
-   사용자 상태 요약 코멘트 제공

### 시각화 리포트

-   주간·월간 감정 변화 그래프
-   감정·이모지 통계 자동 집계
-   감정 패턴 하이라이트 제공

### 심리테스트·오늘의 운세

-   심리 성향 또는 간단한 자기 점검 테스트 제공
-   별자리 정보를 활용한 오늘의 운세 안내
-   감정 기록과 별도로 사용자의 상태를 다양한 관점에서 이해할 수 있는 보조 기능

### 커뮤니티

-   게시글 작성·조회·수정·삭제
-   댓글 및 공감 기능

### 계정·프로필

-   이메일/비밀번호 로그인
-   소셜 로그인(Kakao / Naver / Google)
-   프로필 이미지, MBTI, 생일 설정

---

## 4. 서비스 전체 흐름

1. 사용자는 **일기 작성**
2. AI가 **감정 분석** 및 상태 코멘트 생성
3. 감정 변화·이모지 패턴을 **그래프 및 통계로 시각화**
4. 테스트를 통해 **보조적 자기 점검**
5. 분석된 데이터를 기반으로 **피드백 및 인사이트 제공**

---

## 5. 기술 스택

### Frontend

-   :material-react: React, React Router
-   :material-api: Axios 기반 API 통신

### Backend

-   :material-leaf: Spring Boot 3.x
-   :material-shield-check: Spring Security
-   :material-database: MySQL(RDS), Redis

### Infra

-   :material-aws: AWS EC2, S3, RDS
-   :material-docker: Docker, Docker Hub
-   :material-server: Nginx Reverse Proxy + SSL(Certbot)

## 6. 팀 구성 및 역할

프로젝트는 4명이 참여했으며, 기능 단위로 분업하여 개발을 진행했습니다.
저는 **회원관리** 기능을 담당했으며, 팀장으로서 작업 조율과 **최종 배포 환경 구성**을 함께 맡았습니다.

AI 기능

-   일기·게시판 데이터 기반 감정 분석, 프롬프트 설계, 부적절 콘텐츠 필터링 로직 구현

일기 기능

-   일기 CRUD, 감정 점수 기반 차트 시각화, 주·월 단위 패턴 분석 로직 개발

게시판 기능

-   게시글·댓글·공감 기능, 페이지네이션·검색 기능, 관리자 권한 기반 접근 제어 구현

**회원관리**

-   **이메일·소셜 로그인, JWT·Refresh Token 구조, 프로필 관리, S3 이미지 처리**

**배포**

-   **AWS EC2, Docker, Nginx, Redis, S3, SSL 기반 배포 환경 구성 및 운영**
