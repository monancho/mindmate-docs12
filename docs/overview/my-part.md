# 구현한 주요 기능 정리 (My Role)

> 본 문서는 MindMate 프로젝트에서 제가 직접 담당한 기능을 **기술적 관점**에서 정리한 문서입니다. 인증 시스템, 소셜 로그인, 프로필 이미지 관리, Redis 기반 > 토큰 처리, 그리고 AWS 배포까지 전체적인 회원관리/운영 구조를 설계·구현했습니다.

---

## 1. 인증 시스템 구현 (JWT + Refresh Token + Redis)

### 담당 범위

-   이메일/비밀번호 회원가입 및 로그인
-   Access Token / Refresh Token 발급 구조 구축
-   Redis 기반 Refresh Token 저장 및 TTL 관리
-   토큰 재발급 흐름 설계 및 예외 처리

### 핵심 구현 포인트

-   Access Token은 **단기 인증**, Refresh Token은 **장기 인증** 역할로 분리
-   Refresh Token은 Redis에 `userId:token` 형태로 저장하며 타임아웃(TTL) 적용
-   토큰 만료 시 Axios Interceptor를 통해 **자동 재발급 처리**
-   잘못된 Refresh Token 사용 시 전체 로그아웃 처리

### 기술적 성과

-   서버 부하를 줄이면서 토큰 보안성을 강화하는 구조 완성
-   장애 상황(만료/위조/중복 요청)에 대한 예외 처리가 안정적

---

## 2. 소셜 로그인 (Kakao / Naver / Google OAuth)

### 담당 범위

-   3종 OAuth 로그인 전체 플로우 구현
-   소셜 프로바이더 Access Token 처리
-   신규/기존 회원 분기 처리 로직 구현
-   소셜 탈퇴 및 재연동 시 예외 처리

### 핵심 구현 포인트

-   각 플랫폼마다 상이한 OAuth 규격을 **하나의 통합 구조**로 맞춤 설계
-   콜백 URL ↔ Frontend ↔ Backend 간 인증 코드 전달 과정 조율
-   소셜 로그인 시 이메일 충돌·중복 가입 처리

### 기술적 성과

-   로그인 성공률 상승
-   재로그인, 연동 해제 등 복잡한 흐름 정상 처리

---

## 3. 프로필 이미지 & S3 관리

### 담당 범위

-   프로필 이미지 업로드·변경·삭제 API 구현
-   이미지 파일을 S3에 저장 및 URL 관리
-   회원탈퇴 시 해당 이미지의 S3 객체 삭제
-   업로드 과정에서 이미지 압축 처리 적용

### 핵심 구현 포인트

-   이미지 업로드는 **Multipart/FormData → S3 업로드 후 URL 반환** 구조
-   이미지 품질 유지와 용량 절감을 위해 **프론트와 백엔드에서 이미지 압축 처리** 수행
-   기존 이미지가 있을 경우 자동 삭제하여 S3 저장 비용 최적화

### 기술적 성과

-   안정적인 이미지 업로드/삭제 흐름 확보
-   압축 처리로 이미지 로딩 속도 개선 및 저장 공간 절약
-   S3 객체 관리 자동화로 운영 편의성 향상

---

## 4. 프론트 인증 연동 (Axios Interceptor)

### 담당 범위

-   Access Token 자동 첨부 설정
-   토큰 만료 시 요청 큐 처리(isRefreshing)
-   Refresh Token 재발급 성공 시 대기 중 요청 재시도
-   실패 시 전체 로그아웃 & 초기화

### 핵심 구현 포인트

-   401 발생 시 인터셉터가 **자동으로 재발급 요청을 가로채는 구조**
-   동시 만료 방지를 위해 요청 큐(queue) 구조 구현
-   토큰 재발급 실패 시 사용자를 안전하게 로그아웃 처리

### 기술적 성과

-   사용자 경험(UX) 크게 개선
-   인증 오류로 인한 요청 실패율 감소

---

## 5. AWS 기반 배포 & 운영 환경 구축

### 담당 범위

-   EC2 서버 구성 및 환경 세팅
-   Docker 기반 백엔드 이미지 빌드·배포
-   Nginx Reverse Proxy + SSL(Certbot) 설정
-   Redis, RDS(MySQL), S3 연동
-   서비스 도메인 적용(Route53)

### 핵심 구현 포인트

-   EC2 → Docker Compose 기반 멀티 컨테이너 운영
-   Nginx로 `/api` 라우팅 및 HTTPS 적용
-   백엔드 컨테이너 자동 재시작 정책
-   도메인 연결 시 CloudFront를 활용한 S3 정적 호스팅 구성

### 기술적 성과

-   안정적인 프로덕션 운영 환경 구축
-   SSL 기반 HTTPS 환경으로 보안 강화
-   팀 전체가 동일 환경에서 개발/테스트할 수 있도록 가이드 제작

---

## 6. 팀 내 역할 (Team Lead)

### 수행 역할

-   개발 일정 관리 및 기능 통합
-   팀원 코드 리뷰 및 분기 관리
-   배포 및 운영 문제 대응
-   문서화 및 기술 가이드 제공

### 결과

-   프로젝트 전체 흐름을 안정적으로 유지
-   기능 충돌 최소화 및 환경 변수(.env) 분리를 통한 보안 강화와 팀원별 배포 용이성 확보
-   팀 내 기술 공유 체계 확립
