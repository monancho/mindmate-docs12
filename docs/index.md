## 개요

MindMate는 감정 기록과 AI 분석을 결합한 개인화 자기관리 서비스입니다.
사용자는 일기를 작성하면 시스템이 감정 정보를 자동으로 추출하고, 이를 시각적으로 정리하여 자신의 감정 패턴을 쉽게 이해할 수 있도록 돕습니다.

| 항목        | 내용                                   |
| ----------- | -------------------------------------- |
| 서비스 목적 | 감정 일기 + AI 분석으로 감정 패턴 파악 |
| 주요 대상   | 자기관리·감정 기록 사용자              |
| 핵심 기능   | 일기 작성, 감정 분석, 통계 시각화      |

본 문서는 프로젝트 협업 과정에서 제가 담당한 기능 구현을 중심으로 기술적 내용을 정리합니다.
<br>

## 프로젝트 배경 및 목표

이 프로젝트는 학원 수업에서 배운 기술을 실제 서비스 형태로 구현해보고, 팀 협업 경험을 쌓는 것을 목표로 진행되었습니다.
기획부터 개발, 배포까지 전 과정을 직접 수행하며 실무형 기술·협업 능력을 확보하는 것을 핵심 목표로 설정했습니다.

| 목표 종류 | 내용                           |
| --------- | ------------------------------ |
| 기술 목표 | 실서비스 수준 기능 개발        |
| 협업 목표 | 4인 기능 단위 협업 경험        |
| 배포 목표 | 전체 흐름(기획→배포) 직접 경험 |

<br>

## 담당 역할

프로젝트는 4명이 참여했으며, 프론트엔드·백엔드 역할을 명확히 나누기보다 기능 단위로 분담하는 방식으로 개발을 진행했습니다.  
각 기능 영역은 다음과 같이 구성되었으며, 저는 이 중 **회원관리 기능 전반과 최종 배포 환경 구성**을 담당했습니다.

| 기능 영역         | 담당 내용                                                                   |
| ----------------- | --------------------------------------------------------------------------- |
| AI 기능           | 일기·게시판 기반 감정 분석, 프롬프트 설계, 부적절 콘텐츠 필터링             |
| 일기 기능         | 일기 CRUD, 감정 점수 기반 차트 시각화, 주·월 감정 패턴 분석                 |
| 게시판 기능       | 게시글·댓글·공감, 검색·페이지네이션, 관리자 권한 제어                       |
| **회원관리 (MY)** | **이메일·소셜 로그인, JWT·Refresh Token 구조, 프로필 관리, S3 이미지 처리** |
| **배포 (MY)**     | **AWS · Docker · Nginx · Redis · S3 · SSL · 전체 서비스 배포 구성**         |
