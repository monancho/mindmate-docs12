# MindMate 기술 문서

## 개요

MindMate는 감정 기록과 AI 기반 분석을 중심으로 한 개인화 자기관리 서비스로, 일기를 통해 감정을 자동 정리하고 시각화해주는 웹 애플리케이션입니다.

## 배경 및 목표

MindMate 프로젝트는 학원에서 배운 내용을 실제 서비스 형태로 직접 구현해보고, 팀 프로젝트 경험을 쌓기 위해 진행했습니다. 기획부터 개발, 배포까지 전 과정을 팀원과 함께 구성하는 것을 목표로 했으며, 이 문서는 협업 프로젝트 중 제가 담당한 기능 구현 중심으로 정리한 기술 문서입니다.

## 담당 역할

프로젝트는 4명이 함께 진행했으며, 프론트엔드와 백엔드를 명확히 나누기보다 기능 단위로 작업을 분담하는 방식으로 진행했습니다. 기능은 AI, 일기, 게시글, 회원관리로 크게 나뉘었고, 저는 이 중 회원관리 영역을 담당하고, 최종 배포까지 함께 진행하는 역할을 맡았습니다.

제가 맡은 주요 개발 범위는 다음과 같습니다:

- 인증 시스템 구현<br>
  JWT·Refresh Token·Redis 기반의 인증 및 토큰 관리 구조 구성<br><br>

- 소셜 로그인 연동<br>
  카카오·네이버·구글 OAuth 전체 플로우 개발<br><br>

- 프로필 이미지 & S3 관리<br>
  이미지 업로드·변경·삭제 및 회원탈퇴 시 S3 자원 정리<br><br>

- 프론트 인증 연동<br>
  Axios 인터셉터로 토큰 저장·만료·자동 재발급 처리<br><br>

- 배포 환경 구성<br>
  AWS EC2, Docker, Nginx, Redis, S3, Certbot을 이용한 서비스 배포 환경 구축

## 문서 구성

- 프로젝트 개요 및 전체 구조
- 회원관리(인증·프로필·소셜 로그인) 구현 상세
- 배포 및 인프라 구성(AWS·Docker·Nginx·Redis·S3)
- 트러블슈팅 정리 및 회고

문서는 위 구성에 따라, 제가 담당한 기능을 중심으로 단계별로 설명합니다.
